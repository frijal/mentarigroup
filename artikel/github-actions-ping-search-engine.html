Otomatis "Ping" Search Engine via GitHub Actions

Beri tahu Google & Bing setiap kali websitemu update, biar makin cepat diindeks!

Setelah membahas soal Webhooks, muncul ide: "bisakah kita pakai ini untuk mengirim file sitemap.xml atau rss.xml ke search engine?". Pemikiran ini sangat bagus dan konsepnya sudah tepat! Tapi, caranya sedikit berbeda.

Kita tidak bisa mengirimkan file XML-nya secara langsung. Namun, kita bisa melakukan sesuatu yang lebih simpel dan efektif, yaitu **memberi tahu** Google atau Bing bahwa file kita sudah di-update.

Cara Kerja yang Benar: "Ping", Bukan Mengirim File

Search engine menyediakan sebuah alamat URL khusus yang bisa kita "colek" atau "ping" setiap kali ada pembaruan. Saat URL ini diakses, bot mereka akan tahu dan menjadwalkan kunjungan ulang ke sitemap kita untuk memeriksa konten baru.

Google: https://www.google.com/ping?sitemap=[URL_LENGKAP_SITEMAP]
Bing & Yandex: https://www.bing.com/ping?sitemap=[URL_LENGKAP_SITEMAP]

Kamu hanya perlu mengganti [URL_LENGKAP_SITEMAP] dengan alamat publik sitemap kamu, contohnya https://frijal.pages.dev/sitemap.xml.

Nah, di sinilah ide otomatisasi kamu jadi sangat berguna. Kita bisa membuat proses "ping" ini berjalan otomatis setiap kali kita selesai mempublikasikan konten baru. Cara terbaik dan termudah di GitHub adalah dengan **GitHub Actions**.

Alur kerjanya persis seperti yang kamu bayangkan:

  1. Kejadian: Kamu melakukan push ke branch main yang berisi artikel baru (yang juga memperbarui file sitemap.xml dan rss.xml).
  2. Pemicu: Workflow GitHub Actions berjalan secara otomatis setelah proses *deploy* selesai.
  3. Tindakan: Sebuah langkah khusus di dalam workflow akan menjalankan perintah curl untuk "memanggil" URL ping dari Google dan Bing.

Contoh Langkah di GitHub Actions

Kamu bisa menambahkan langkah sederhana ini di akhir *job deploy* kamu. Langkah ini akan mengirimkan "ping" ke Google dan Bing untuk memberitahu bahwa sitemap.xml dan rss.xml kamu sudah diperbarui.

      # ... langkah-langkah deploy Anda sebelumnya ...

      - name: ðŸ”” Ping Search Engines for Updates
        run: |
          # Ping sitemap utama
          curl -s "https://www.google.com/ping?sitemap=https://frijal.pages.dev/sitemap.xml"
          curl -s "https://www.bing.com/ping?sitemap=https://frijal.pages.dev/sitemap.xml"
          
          # Ping RSS feed utama (opsional, tapi bagus)
          curl -s "https://www.google.com/ping?sitemap=https://frijal.pages.dev/rss.xml"
          curl -s "https://www.bing.com/ping?sitemap=https://frijal.pages.dev/rss.xml"

Catatan kecil: Perintah curl -s menggunakan flag -s yang artinya "silent" (senyap), jadi log di GitHub Actions kamu tidak akan dipenuhi dengan output detail dari proses `curl`, membuatnya lebih bersih dan rapi.

Kesimpulan

Jadi, dengan menambahkan beberapa baris sederhana di workflow GitHub Actions, kamu bisa memastikan search engine besar langsung tahu setiap kali kamu punya konten baru. Ini cara yang sangat efektif untuk mempercepat proses *indexing* dan membuat artikelmu lebih cepat muncul di hasil pencarian!
