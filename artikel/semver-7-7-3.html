📦 semver@7.7.3 — Kalkulator Versi untuk Node.js

Semver adalah paket npm yang dipakai luas untuk parsing, membandingkan, dan mengelola versi software sesuai aturan Semantic Versioning. Cocok untuk dependency management, CI/CD, sistem plugin, dan hal-hal yang butuh akurasi versi.

Dipublikasikan: 27 Okt 2025 npm Node.js Semantic Versioning
Ringkas manfaat:
  • 🧠 Akurat: parsing & validasi versi yang mengikuti SemVer.
  • 🛠 Praktis: bandingkan versi, cek range, dan increment otomatis.
  • 🔒 Dipakai npm: memastikan dependensi sesuai constraint di package.json.
  • ⚙️ CI/CD: guardrail versi sebelum publish atau deploy.

Apa itu semver dan kenapa penting?

Semver (Semantic Versioning) adalah cara konsisten memberi versi pada software: MAJOR.MINOR.PATCH. Paket semver@7.7.3 di npm menyediakan fungsi-fungsi untuk parsing, validasi, komparasi, dan penentuan kecocokan versi terhadap sebuah range. Ini krusial saat aplikasi kamu bergantung pada banyak library, karena keliru satu versi saja bisa bikin build rusak atau fitur tidak kompatibel.

Tabel ringkas fitur semver

Fitur / Fungsi Contoh Kode Hasil / Kegunaan
Parse versi semver.parse('1.2.3') Objek { major:1, minor:2, patch:3 }
Bandingkan versi semver.gt('1.2.3','1.2.2') true → cek apakah versi lebih besar
Cek kesesuaian range semver.satisfies('1.5.0','^1.4.0') true → cocok dengan range ^1.4.0
Increment versi semver.inc('2.3.4','minor') 2.4.0 → naik versi minor
Validasi string versi semver.valid('1.2.3') 1.2.3 (valid) atau null jika tidak valid
Maksimum/Minimum semver.maxSatisfying(['1.2.3','1.5.0'],'^1.0.0') 1.5.0 → versi tertinggi sesuai range
Cek prerelease semver.prerelease('1.2.3-beta.2') ['beta',2] → info label & nomor prerelease

Contoh penggunaan cepat

// Instalasi
// npm install semver@7.7.3

const semver = require('semver');

// Bandingkan versi
console.log(semver.gt('1.2.3', '1.2.2'));  // true

// Cek apakah versi cocok dengan range
console.log(semver.satisfies('1.5.0', '^1.4.0'));  // true

// Naikkan versi patch/minor/major
console.log(semver.inc('2.3.4', 'patch')); // 2.3.5
console.log(semver.inc('2.3.4', 'minor')); // 2.4.0
console.log(semver.inc('2.3.4', 'major')); // 3.0.0

// Validasi versi
console.log(semver.valid('1.2.3')); // '1.2.3'
console.log(semver.valid('1.2'));   // null

// Pilih versi tertinggi yang sesuai range
console.log(semver.maxSatisfying(['1.0.0','1.5.1','1.5.0'], '^1.5.0')); // '1.5.1'

Kapan semver dipakai?

  • Library npm: menentukan range dependensi di package.json agar kompatibel.
  • Tool CI/CD: memastikan versi baru lebih tinggi dari sebelumnya sebelum publish/deploy.
  • Sistem plugin: cek kompatibilitas plugin terhadap core app berdasarkan range versi.
  • Rilis terstruktur: disiplin rilis dengan MAJOR/MINOR/PATCH untuk memudahkan tracking perubahan.

Semver membantu tim menjaga konsistensi dan mencegah “dependency hell”. Kombinasikan dengan test otomatis agar upgrade versi tetap aman.

Info teknis singkat

  • Versi paket: 7.7.3
  • Lisensi: ISC
  • Dependensi: tidak ada (ringan)
  • Kompatibilitas: Node.js (CommonJS). Untuk ESM, gunakan import semver from 'semver' + bundler sesuai.

Tips praktis

  • Range caret (^): aman untuk minor/patch, tapi hindari di paket kritis bila butuh stabilitas maksimal.
  • Lockfile: pastikan package-lock.json/pnpm-lock.yaml dikomit untuk reproducible build.
  • Prerelease: tandai beta/rc dengan label agar tidak terkunci ke rilis final—1.2.0-rc.1.
