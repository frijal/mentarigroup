Workflow 'Cari & Ganti' Otomatis via GitHub Actions

Alat canggih untuk mengedit file secara massal di repositori Anda.

Pernahkah kamu perlu mengganti satu kata atau kalimat yang sama di puluhan file berbeda? Misalnya, mengupdate tahun copyright di footer, mengubah link, atau me-refactor nama variabel. Tentu melelahkan jika dilakukan satu per satu.

Dengan **GitHub Actions**, kita bisa membuat sebuah *workflow* yang berfungsi sebagai alat **"Cari dan Ganti" (Find and Replace)** otomatis untuk seluruh repositori kita. Semuanya bisa dijalankan secara manual langsung dari halaman GitHub.

Bagaimana Cara Pakainya?

  1. Pergi ke tab "Actions" di repositori GitHub kamu.
  2. Di menu sebelah kiri, pilih workflow "ganti isi teks dalam file".
  3. Klik tombol "Run workflow".
  4. Sebuah formulir akan muncul. Isi semua kolom yang dibutuhkan:
    • Teks yang mau diganti (search): Kata atau kalimat yang ingin kamu cari.
    • Teks pengganti (replace): Kata atau kalimat baru sebagai penggantinya.
    • Folder target (folder): Folder spesifik tempat pencarian (misal: docs). Biarkan . untuk mencari di seluruh repo.
    • Polanya file (globs): Tipe file yang akan diubah, pisahkan dengan koma (misal: **/*.html,**/*.md).
  5. Klik tombol hijau "Run workflow". Selesai!

Kode Workflow Lengkap

Ini adalah kode lengkap yang bisa kamu simpan di repositori GitHub kamu, misalnya di file .github/workflows/find-replace.yml.

üìÑ .github/workflows/find-replace.yml 

name: ganti isi teks dalam file

on:
  workflow_dispatch:
    inputs:
      search:
        description: 'Teks yang mau diganti (case-sensitive)'
        required: true
        default: 'makan siang bersama'
      replace:
        description: 'Teks pengganti'
        required: true
        default: 'mandi pagi sendirian'
      folder:
        description: 'Folder target (misal: content, src). Kosong = root repo'
        required: false
        default: '.'
      globs:
        description: 'Polanya file (misal: **/*.txt,**/*.html)'
        required: false
        default: '**/*.txt,**/*.html'

jobs:
  replace:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Jalankan penggantian string di folder terpilih
        run: |
          TARGET="${{ github.event.inputs.folder }}"
          SEARCH_TERM="${{ github.event.inputs.search }}"
          REPLACE_TERM="${{ github.event.inputs.replace }}"
          
          # Ubah koma menjadi spasi untuk loop
          read -ra GLOBS <<< "${{ github.event.inputs.globs//,/ / }}"

          for g in "${GLOBS[@]}"; do
            echo "üîé Proses pola: '$g' di folder '$TARGET'"
            # Gunakan find -path untuk mencocokkan pola **/*
            find "$TARGET" -path "$g" -type f -print0 | while IFS= read -r -d '' file; do
              # Gunakan perl untuk find & replace yang aman
              perl -pi -e 's#\Q$ENV{SEARCH_TERM}\E#$ENV{REPLACE_TERM}#g' "$file"
            done
          done

      - name: Commit & push perubahan
        run: |
          git config user.name "${{ github.actor }}"
          git config user.email "${{ github.actor }}@users.noreply.github.com"
          git add -A
          # Gunakan || true agar tidak error jika tidak ada perubahan
          git commit -m "chore: ganti '${{ github.event.inputs.search }}' -> '${{ github.event.inputs.replace }}'" || true
          git push

Bedah Singkat Kode

  • on: workflow_dispatch: inputs:: Bagian ini yang membuat formulir interaktif saat kita menekan "Run workflow".
  • Jalankan penggantian...: Di langkah ini, kita menggunakan kombinasi perintah find untuk mencari file berdasarkan pola yang kita masukkan, dan perl untuk melakukan penggantian teks di dalam setiap file yang ditemukan. Perl dipilih karena sangat andal dalam menangani karakter-karakter aneh yang mungkin ada di teks.
  • Commit & push...: Setelah selesai, skrip akan otomatis membuat *commit* baru dengan pesan yang jelas dan mendorongnya ke repositori. Jadi, semua perubahan langsung tersimpan.
