Kenapa Sih File .yml GitHub Actions Sering Pake 'setup-node'?

Singkatnya: Kamu **wajib** pake actions/setup-node kalau pekerjaan (job) kamu butuh Node.js atau NPM buat jalan. Kalau nggak butuh, ya nggak usah.

Apa Itu "Runner"?

Biar gampang, bayangin tiap kali *workflow* kamu jalan, GitHub ngasih kamu satu "komputer virtual baru yang masih fresh". Komputer ini disebut Runner.

Komputer ini bisa jadi pake Linux, Windows, atau macOS. Walaupun dibilang "fresh", GitHub udah nginstal banyak software umum di dalamnya, termasuk beberapa versi Node.js, Python, Go, dll.

Kalo Udah Ada, Kenapa Tetep Perlu 'setup-node'?

Nah, ini dia intinya. Walaupun Node.js mungkin udah ada di *Runner*, kita tetep perlu actions/setup-node karena 3 alasan penting:

  1. Memastikan Versi yang Tepat. Proyek kamu mungkin dibuat pake Node.js versi 20. Eh, ternyata *Runner* yang kamu dapet kebetulan pake Node.js versi 16 sebagai default-nya. Hasilnya? Perintah npm install kamu bisa gagal total karena versi nggak cocok. Ambyar, kan.
  2. Konsistensi Itu Kunci. Dengan nambahin with: { node-version: 20 }, kamu "maksa" si *Runner* buat pake Node.js versi 20, nggak peduli apa pun versi default di komputer itu. Ini bikin *workflow* kamu stabil dan gampang diprediksi.
  3. Mengatur "PATH" Sistem. Ini penting banget. *Action* ini otomatis "ngasih tau" si *Runner* di mana letak file `node` dan `npm` yang bener. Jadi, pas kamu jalanin perintah npm install atau npm run build di langkah selanjutnya, si *Runner* nggak bingung nyariin file-nya di mana.

Bedah Kodenya

Jadi, pas kamu nulis kode kayak di gambar ini, sebenernya kamu lagi ngapain sih?

Itu artinya kamu lagi ngasih instruksi ke GitHub. Biar gampang nyalinnya, ini teks kodenya: 

- name: Setup Node.js
  uses: actions/setup-node@v4
  with:
    node-version: 20
    cache: 'npm' # <-- Tambahan keren!
  • name: Setup Node.js: Ini cuma label. Judul buat kamu baca di log, biar nggak bingung.
  • uses: actions/setup-node@v4: Ini perintah intinya. "GitHub, tolong pake skrip sakti bernama 'setup-node' (versi 4) bikinan tim 'actions'."
  • with:: Ini adalah "opsi" atau pengaturan buat skrip sakti tadi.
  • node-version: 20: "Tolong cariin Node.js versi 20 dan jadikan versi yang aktif."
  • cache: 'npm': Ini bonus mantap. "Sekalian tolong *cache* folder node_modules ya, biar npm install di proses selanjutnya ngebut!"

Kapan Nggak Perlu Dipake?

Ya gampang, kalau *workflow* kamu sama sekali nggak nyentuh Node.js atau NPM.

  • Lagi ngerjain proyek Python? Kamu bakal pake actions/setup-python@v5.
  • Lagi ngerjain proyek Go? Kamu bakal pake actions/setup-go@v5.
  • Cuma mau nge-build Docker image? Kamu mungkin cuma pake docker/build-push-action@v6.
