Workflow Otomatis: Ping Sitemap & Feeds via GitHub Actions

Beri tahu Google & Bing setiap kali websitemu update, biar makin cepat diindeks!

Setelah situs kita di-deploy, langkah selanjutnya yang tidak kalah penting adalah memberitahu *search engine* seperti Google dan Bing bahwa ada konten baru. Tujuannya? Tentu saja agar mereka lebih cepat datang dan meng-indeks halaman baru kita.

Daripada melakukannya manual, kita bisa membuat proses ini sepenuhnya otomatis menggunakan **GitHub Actions**. Ini adalah workflow yang sudah disempurnakan untuk melakukan "ping" ke berbagai layanan setiap hari.

Kode Workflow Lengkap

Ini adalah kode lengkap yang bisa kamu simpan di repositori GitHub kamu, misalnya di file .github/workflows/ping-feeds.yml.

📄 .github/workflows/ping-feeds.yml 

name: Ping Feeds & Sitemap

on:
  workflow_dispatch:    # Bisa dijalankan manual
  schedule:
    - cron: "0 6 * * *"    # Otomatis tiap hari jam 06:00 UTC

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: 🔔 Ping Feeds ke Ping-o-Matic
        run: |
          BASE_URL="https://frijal.pages.dev"
          BLOG_TITLE="Frijal Pages"
          FEEDS=(
            "feed-catatan-and-opini-sosial.xml"
            "feed-kuliner-gaya-hidup-and-kesehatan.xml"
            "feed-lainnya.xml"
            "feed-linux-and-open-source.xml"
            "feed-multimedia-and-editing.xml"
            "feed-sejarah-and-religi.xml"
            "feed-teknologi-web-ai-and-umum.xml"
            "rss.xml"
          )
          echo "--- Pinging Ping-o-Matic for ${#FEEDS[@]} feeds ---"
          for feed_file in "${FEEDS[@]}"; do
            FEED_URL="$BASE_URL/$feed_file"
            echo "Pinging: $FEED_URL"
            curl -s -o /dev/null -X POST \
              --data-urlencode "title=$BLOG_TITLE" \
              --data-urlencode "blogurl=$BASE_URL/" \
              --data-urlencode "rssurl=$FEED_URL" \
              https://pingomatic.com/ping/
          done
          echo "--- Ping-o-Matic Pings Sent ---"

      - name: 📍 Ping Sitemap & Main RSS ke Google & Bing
        run: |
          BASE_URL="https://frijal.pages.dev"
          SITEMAP_URL="$BASE_URL/sitemap.xml"
          RSS_URL="$BASE_URL/rss.xml"

          echo "--- Pinging Google ---"
          echo "Pinging Sitemap: $SITEMAP_URL"
          curl -s "https://www.google.com/ping?sitemap=$SITEMAP_URL"
          echo "Pinging RSS Feed: $RSS_URL"
          curl -s "https://www.google.com/ping?sitemap=$RSS_URL"
          
          echo "--- Pinging Bing (+ Yandex) ---"
          echo "Pinging Sitemap: $SITEMAP_URL"
          curl -s "https://www.bing.com/ping?sitemap=$SITEMAP_URL"
          echo "Pinging RSS Feed: $RSS_URL"
          curl -s "https://www.bing.com/ping?sitemap=$RSS_URL"
          
          echo "--- Google & Bing Pings Sent ---"

Bedah Workflow Baris per Baris

Mari kita pahami setiap bagian dari workflow di atas.

1. Pemicu (Triggers)

Bagian on: menentukan kapan workflow ini akan berjalan.

  • workflow_dispatch: Ini memungkinkan kita menjalankan workflow secara manual dari tab "Actions" di GitHub. Berguna untuk testing atau jika kita butuh melakukan ping di luar jadwal.
  • schedule: - cron: "0 6 * * *" Ini adalah pemicu utamanya. Workflow akan berjalan **otomatis setiap hari pada jam 06:00 UTC**. Kamu bisa sesuaikan waktunya sesuai kebutuhan.

2. Langkah 1: Ping ke Ping-o-Matic 🔔

Langkah pertama ini bertujuan untuk jangkauan yang lebih luas. **Ping-o-Matic** adalah layanan gratis yang akan meneruskan "ping" kita ke berbagai layanan aggregator feed lainnya, bukan hanya search engine besar.

  • Skrip ini membuat daftar semua file feed per kategori (FEEDS).
  • Kemudian ia melakukan *looping*, mengirimkan *request* POST ke Ping-o-Matic untuk setiap feed.
  • Kita menggunakan curl untuk mengirim data seperti judul blog dan URL feed yang sedang diperbarui.

3. Langkah 2: Ping ke Google & Bing 📍

Ini adalah langkah yang paling penting. Kita memberitahu langsung dua search engine terbesar bahwa situs kita punya pembaruan.

  • Untuk efisiensi, kita hanya perlu me-ping file utama, yaitu sitemap.xml (yang berisi semua link halaman) dan rss.xml (feed utama). Google dan Bing cukup pintar untuk mengikuti link dari sana.
  • Lagi-lagi kita menggunakan curl -s untuk mengirim *request* ke URL ping milik Google dan Bing, dengan parameter URL sitemap kita.

Gambar di atas menunjukkan log dari GitHub Actions saat workflow ini berhasil dijalankan. Terlihat jelas setiap langkah dan URL mana saja yang berhasil di-"ping". Semuanya berjalan mulus dan otomatis!
