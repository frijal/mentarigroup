Cara Membuat API Cloudflare

Sering pusing harus bolak-balik buka dashboard Cloudflare cuma buat hapus cache, ubah DNS, atau deploy project? Tenang, ada cara otomatisnya pakai Cloudflare API Token. Ini dia panduan lengkap dan santainya, langkah demi langkah.

https://dash.cloudflare.com/profile/api-tokens

Nanti di halaman itu, cari dan klik tombol biru yang tulisannya ‚ÄúCreate Token‚Äù.

Di sini, Cloudflare ngasih banyak pilihan. Biar gampang dan aman, kamu bisa pakai template yang udah ada. Misalnya, kalau cuma buat hapus cache (purge), pilih aja ‚ÄúZone: Cache Purge‚Äù.

Tapi kalau butuh yang lebih spesifik, kamu bisa pilih "Create Custom Token". Ini contoh setting izin (permission) yang sering dipakai:

- Zone ‚Üí Cache Purge ‚Üí Edit  (Untuk hapus cache)
- Zone ‚Üí Zone ‚Üí Read        (Untuk baca info zona)

Setelah atur semua izin dan klik "Create Token", Cloudflare bakal ngasih kamu satu kode API yang panjang banget. Kayak gini contohnya:

CF_API_TOKEN = 1a2b3c4d5e6f7g8h9iJkLmNoPqRsTuVwXyZ...

PERHATIAN: Langsung salin (copy) dan simpan token ini di tempat yang aman (kayak password manager). Token ini cuma bakal muncul SATU KALI AJA. Kalau halaman ini kamu refresh atau tutup, tokennya hilang selamanya dan kamu harus bikin baru lagi.

Nah, token ini paling sering dipakai buat otomatisasi di GitHub Actions. Biar aman, jangan taruh tokennya langsung di kode kamu. Simpan di "Secrets".

Masuk ke repositori GitHub kamu, terus ke:

Settings ‚Üí Secrets and variables ‚Üí Actions ‚Üí New repository secret

Lalu isi formulirnya:

  • Name: CF_API_TOKEN
  • Value: tempel_token_rahasia_kamu_di_sini

Mau mastiin tokennya berfungsi? Kamu bisa tes pakai curl di terminal kamu. Ganti <TOKEN_KAMU> dengan token yang tadi kamu salin:

curl -X GET "https://api.cloudflare.com/client/v4/user/tokens/verify" \
     -H "Authorization: Bearer <TOKEN_KAMU>" \
     -H "Content-Type: application/json"
